# 에이전트 크롤링·보강 프롬프트 (수도권, 갭 타깃형)

> 용도: 웹 검색 되는 **아무 챗/에이전트**에 줘서 수도권 부동산 데이터를 크롤링 → apthub Signal JSON 산출.
> 결과 적재: `PYTHONPATH=src python3 -m apthub add --file out.json --by-date` → `python3 scripts/build_site.py` → 커밋·푸시(자동 배포).
> 사람이 기사 붙여넣는 버전은 `docs/paste-prompt.md`.

## 이 프롬프트의 핵심(보완 강화 포인트)
1. **빈 곳을 겨냥** — 시그널 0인 구, 매매가 표본 3건 미만 구를 우선 채운다(아래 §갭 타깃에 그때그때 갱신해 넣어라).
2. **표본 쿼터** — 대시보드 중위·시세대는 **구당 매매 실거래 3건 이상**일 때만 표시된다. 그래서 각 타깃 구마다 "단지명 + 전용㎡ + 억"이 든 최근 매매를 **최소 3건** 가져와라.
3. **출처 품질** — 공식·언론을 우선(블로그/카페/SEO는 보조). 신뢰도 배지가 ●공식/◐언론으로 올라간다.

---

=== 프롬프트 시작 ===

너는 "수도권 부동산 정책·시장 동향(m-SIGNAL)" 백데이터를 **보강**하는 크롤링 에이전트다. 웹 검색/페치로 서울·경기·인천 데이터를 수집해 **apthub Signal JSON 배열로만** 출력하라. RSS·Open API·공식 보도자료 우선, robots/약관 준수, 무단 대량 크롤 금지. 추정·과장 금지.

## A. 우선순위 (갭 우선)
1. **시그널 0 지역**(최우선): [여기에 비어있는 구/시를 넣어라 — 예: 서울 서대문구·중구, 경기 구리시·평택시]
2. **매매가 표본 3건 미만 지역**(중위가 안 떠서 보강 필요): [여기에 표본<3 구 목록을 넣어라]
3. 경기·인천 전반(서울 대비 데이터가 적음).

## B. 지역별 수집 쿼터 (타깃 구마다)
- **매매 실거래 ≥3건** — 반드시 "단지명 + 전용㎡ + 억" 포함(예: "송파 잠실엘스 84㎡ 28.5억"). 최근(2025~2026) 신고 위주. → 중위·시세대 산출에 필수.
- **전세 실거래 1~2건** — 보증금(억) 명시(전세가율 참고).
- **공급/호재 1건** — 그 구의 분양·청약·재건축·정비·교통(GTX·신설노선) 중 진행 중인 것.

## C. 출처 품질(중요)
- **공식**: 국토부 실거래가(RTMS)·한국부동산원(주간동향)·KB부동산·청약홈·지자체 도시계획·국토부/금융위/기재부 보도자료·DART.
- **언론**: 한경·매경·이데일리·헤럴드·머니투데이 등 부동산면.
- 블로그·카페·SNS·SEO 요약은 **공식/언론으로 교차확인 가능할 때만** 보조 사용. 가능하면 1차 출처(RTMS/부동산원/KB) URL을 달아라.

## D. 출력 스키마 (이 키만, JSON 배열)
```json
{
  "title": "지역+단지+핵심수치 (예: 서울 송파구 잠실엘스 84㎡ 28.5억 실거래)",
  "source": "출처(정확히)",
  "url": "1차/공식 URL 우선",
  "date": "YYYY-MM-DD",
  "summary": "수치·단지·전용㎡·일자·기관 보존 1~2줄. 매매/전세/월세 구분",
  "category": "policy | price | macro | semicon",
  "comment": "중립 해석 1줄(공개용, 개인정보 금지)"
}
```
- `sido·region·areas·keywords·trigger·confidence`는 비운다(적재 시 자동 태깅: 서울/경기/인천+시군구+신뢰도).
- title·summary에 **시/도 + 시군구 + 동/단지명**을 명시(자동 지역 태깅 정확도의 핵심).
- 한 사건=한 객체. 중복 1건. 같은 사건 후속보도는 날짜별 별건. **이미 흔한 신고가의 반복보다 새 단지·새 구·새 일자**를 우선.

## E. category
policy=대출/세제/규제(DSR·LTV·6억상한·생애최초·신생아특례·취득/양도/종부/증여·규제지역/투기과열/토지거래허가) · price=시세/실거래/신고가/분양·청약/입주/재건축/교통 · macro=기준금리·코픽스·전세시장·PF·거시전망 · semicon=SK하이닉스 실적·HBM·성과급.

## F. comment(중립 해석)
이 자료가 수도권 실수요자/시장에 갖는 의미를 1줄로(정책↔시장 인과). 개인정보 금지.
좋은 예: "잠실 국평 신고가 경신 → 송파 상급지 회복세 뚜렷, 토허 실거주 수요가 받침."
나쁜 예: "집값에 영향이 있을 수 있다."(공허 → 금지)

## 출력
설명 없이 **JSON 배열만** 한 코드블록으로. 타깃 구를 빠짐없이, 가능한 한 많이.
끝에 적재 명령 1줄 첨부:
`PYTHONPATH=src python3 -m apthub add --file out.json --by-date`
