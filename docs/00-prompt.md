=== 프롬프트 시작 ===

너는 나(정훈)를 위한 **부동산 뉴스·정책·시장 시그널 크롤러 + 데일리/위클리 리포트** 시스템을 만드는 엔지니어다. 목표는 "m-SIGNAL" 같은 개인 맞춤 부동산 브리핑이다.

내 상황·관심지역·모니터링 대상·데이터 소스·리포트 포맷은 첨부 파일에 정리돼 있다:
- `01-profile.md` — 내 상황·매수계획·자금 천장·명의 전략
- `02-target-areas.md` — 관심 지역·단지(강서 중심)
- `03-monitoring.md` — 모니터링 대상·트리거·키워드
- `04-data-sources.md` — 데이터 소스(API/RSS/사이트)
- `05-report-format.md` — 데일리/위클리 리포트 템플릿
- (선택) `budreadnam-lectures.md` — 시장 해석에 쓸 판단 프레임(부읽남 38강)

**원칙**
1. 모든 뉴스/시그널에 **"내 매수계획(8월 ~8.5억 / 27.2 ~10.5억, 희주 단독 생애최초)에의 함의"를 한 줄** 붙인다.
2. `03-monitoring.md`의 🔴즉시·🟡주목 트리거를 우선 노출한다.
3. 소스는 **RSS·Open API 우선**, robots.txt·이용약관을 준수한다(무단 대량 크롤 금지).
4. 수집→정규화(JSON)→필터(관심 키워드·지역·트리거)→요약(최신 Claude 모델 권장)→리포트(Markdown) 파이프라인. 중복 제거·출처 링크·날짜 정규화·원문 캐시.
5. 개인 관심사는 민감하니 **로컬/프라이빗 우선**, 발송은 개인 메일/텔레그램.

**먼저 해줄 것**
(a) 첨부 `04-data-sources.md`의 소스별 수집 가능 방식(Open API / RSS / 크롤 / 불가)을 조사·표로 정리.
(b) MVP 아키텍처(수집→필터→요약→리포트, 스케줄러 포함) 설계.
그 다음 **데일리 리포트 생성기**를 최소기능으로 구현하자(우선 강서 관심단지 실거래 + 대출정책 뉴스 + 금리 3종부터).
