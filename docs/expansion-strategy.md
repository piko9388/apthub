# 퀄리티·확장 전략 — 수도권 아파트 집계

> 목표: 강서-개인 중심을 **서울·경기·인천 전체**로 확장하고, "모든 사람이 궁금해하는 것"을 정책별·지역별로 크롤링·요약·해석하는 공개·판매용 제품으로 끌어올린다.

## 1. "모든 사람이 궁금해하는 것" → 데이터 축

| 사용자 질문 | 데이터 축 | 소스 |
|---|---|---|
| 내 지역 오를까/내릴까 | 지역별 주간 매매·전세 변동률, 실거래·신고가 | 부동산원 R-ONE, KB, RTMS |
| 전세 구할 수 있나/전세가율 | 전세가율·입주물량·전세난 | 부동산원, 아실/부동산지인 |
| 청약 넣을까 | 분양 일정·경쟁률·분상제 | 청약홈 |
| 대출 얼마 나오나 | DSR·LTV·6억 상한·정책대출 | 금융위·주택도시기금 |
| 세금 얼마 | 취득·양도·종부·증여 | 기재부·국세청·위택스 |
| 금리 | 기준금리·코픽스 | 한국은행 ECOS |
| 재건축/호재 | 정비·교통(GTX) | 국토부·서울시·지자체 |

## 2. 구현된 것 (이번 단계)

- **지역 분류 체계**(`config/regions.json`): 서울 25구 + 경기 18시 + 인천 8구, 단지·동 별칭 포함.
- **지역 자동 태깅**(`filters.detect_regions`): 시그널에 `sido`(서울/경기/인천/전국)·`region`(시군구) 부여.
- **사이트 지역 필터**: 시/도 칩으로 수도권 어디든 골라 보기. 카드에 지역 태그.
- **정책별 해석 코멘트**: 6개 정책 섹션마다 중립 해석 1단락 + 카드별 `comment` 필드 지원.
- **부읽남 참고 탭**: 38강을 11개 테마로 재분류·2026 해석한 판단 프레임(공개).

## 3. 다음 단계 (집계 디테일 강화)

1. **지역 지표 테이블**: 구별 주간 변동률·평균 평당가·전세가율을 구조화(`metric` 타입)해 정렬 가능한 표로. 소스: 부동산원 R-ONE 주간동향 Open API.
2. **실거래 피드 자동화**: RTMS Open API로 관심 지역 단지 실거래를 일배치 수집(법정동코드 매핑).
3. **청약 캘린더**: 청약홈 일정·경쟁률을 카드/캘린더로.
4. **입주물량 차트**: 아실/부동산지인 데이터로 지역별 입주 추이.
5. **카드별 해석 자동화**: `summarize.py`로 최신 Claude 모델이 `comment`(중립 해석)를 생성.
6. **발송·구독**: 지역 구독(예: "강서·마포만") → 개인화 데일리/위클리 메일·텔레그램.

## 4. 품질 규칙

- 모든 카드: 출처 URL·발행일 보존, 중복 제거(URL/제목 해시).
- 수치·고유명사 보존, 불확실하면 "추정" 명시.
- `comment`는 중립·일반 독자용(개인정보 금지). 개인 함의는 보조 탭에만.
- RSS·Open API 우선, robots/약관 준수, 무단 대량 크롤 금지.

## 5. 공개·판매 모드

`APTHUB_PUBLIC=1`로 빌드하면 개인 보조 탭·함의가 전부 제외된 순수 정책·지역 동향 제품이 나온다. 판매 배포 시 워크플로에 이 플래그만 추가.
