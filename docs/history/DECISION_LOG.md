# APT-SIGNAL 의사결정 로그 (Decision Log)

> 프로젝트 진행 중 내린 핵심 결정과 근거를 시간순으로 기록한다(ADR 성격).
> 코드/데이터의 "왜"를 보존해 컨테이너 리셋·세션 교체에도 맥락이 유지되도록 한다.

---

## D01. 수집은 채팅 수동파싱부터 (Phase 1)
- **결정**: 자동 크롤러를 먼저 만들지 않고, 채팅에 기사를 붙여넣어 Signal JSON으로 파싱하는 방식으로 시작.
- **근거**: robots/약관·차단(403) 리스크 회피, 빠른 검증 루프. 정규화→필터→트리거→리포트는 코드로 자동화.
- **귀결**: `docs/*-crawl-prompt.md` 프롬프트군 + `apthub add --by-date` 적재 파이프라인.

## D02. Signal 단일 스키마로 모든 입력 정규화
- **결정**: 뉴스/실거래/공시/지표를 모두 `Signal` 한 단위로.
- **근거**: 이종 소스를 한 파이프라인(필터·태깅·트리거·리포트)으로 처리.
- **세부**: `make_id = sha1(url + "\n" + title)` — 집계사이트가 같은 랜딩 URL을 여러 거래에 재사용하므로 URL+제목으로 구분.

## D03. 발행일별 분산 저장(by-date)
- **결정**: 시그널을 발행일 파일(`data/seed/…`, 적재 시 `store.add_by_date`)로 분산.
- **근거**: 과거 시점 조회·일자별/주차별 리포트 발행을 위해 날짜가 1차 키.

## D04. 공개 제품으로 피벗 — 개인(정훈/희주)은 보조
- **결정**: 메인 = 비개인화 수도권 부동산 정책·시장 동향. 개인 맞춤은 보조 탭.
- **근거**: 외부 공개·판매 목적. `APTHUB_PUBLIC=1` 빌드시 개인 탭 완전 제외.

## D05. 수도권 전역 커버리지 + 품질 게이트
- **결정**: 서울 25구·경기 18시·인천 8구 분류체계(`config/regions.json`). 매매 중위·시세대는 **구당 표본 3건 이상**일 때만 표시.
- **근거**: 표본 부족 구의 과대/오표기 방지. 신뢰 가능한 지표만 노출.
- **세부**: `detect_regions` 최장 토큰 우선 매칭·소거 → '남동구'→'동구' 부분문자열 오태깅 방지.

## D06. 정적 단일 HTML + GitHub Pages
- **결정**: 외부 의존성 없는 자체완결 `index.html`(인라인 JSON + 바닐라 JS + Leaflet). main 푸시 시 Pages 자동 배포.
- **근거**: 운영 단순화·무료 호스팅. 루트 `index.html`도 출력(브랜치배포 시 README 대신 서빙).

## D07. 출처 신뢰도 배지 (공식/언론/추정)
- **결정**: URL 도메인 → 신뢰도 등급(`TRUST`). 공식(부동산원·한은·국토부 등) > 언론 > 추정.
- **근거**: 자극적·미검증 정보와 1차 통계를 시각적으로 구분.

## D08. 편집형 '종합 동향' 브리핑 추가
- **결정**: 카드 나열식 보완 — 상단에 사람이 작성하는 편집형 브리핑(`config/report.json`), 핵심 수치는 빌드시 라이브 치환.
- **근거**: "정보는 많은데 너무 병렬식" 피드백. 흐름·해석 제공.

## D09. 브랜딩 APT-SIGNAL
- **결정**: m-SIGNAL → APT-SIGNAL(하이픈). 헤더/푸터/리포트 제목 통일.

## D10. 시계열 묶음 단위: 주 → 월 (이후 주/일 드릴다운으로 확장)
- **결정(1차)**: 주차별 묶음이 너무 잘게 쪼개져, 월별 묶음으로 전환 + '한눈에' 기준기간을 핵심 구간(상위 5% 아웃라이어 절단)으로 표기.
- **근거(1차)**: 2024~2026 전체 범위는 너무 길어 상승/하락 추세가 묻힘.
- **결정(2차, D14)**: 리포트 발행 축은 **주차별 메인 + 일자별 드릴다운**(주간 리포트가 일자별을 포함). 월별 플로우는 모니터링의 장기 추세로 보존.

## D11. 2트랙 데이터 모델 (뉴스 ↔ 지표)
- **결정**: `Signal.kind = news|data`. 지표(`metric/value/unit`)는 자동 data 트랙, 실거래 중위 표본에서 제외.
- **근거**: "뉴스는 자극적·단편적" → 공식 정량 지표를 따로 모아 **정합(reconcile)**, 뉴스 과장 여부를 지표로 검증.

## D12. 동향 모니터링(정합) 뷰
- **결정**: 좌측 📊 — 금리·거시/지역 가격·거래 지표 타일 + 뉴스↔지표 정합 카드(지표 방향 vs 뉴스 과열어/약세어 → 과장 판정).
- **근거**: 실제 추세 모니터링. 지표가 보합인데 뉴스가 과열이면 "뉴스 과장 주의".

## D13. 표준 metric 사전 (지표 섞임 방지)
- **결정**: 부동산원 월간 vs 주간 vs KB는 metric 이름을 분리(`매매가격지수 변동률`/`주간 매매변동률`/`KB 매매변동률`). 거래량도 신고일(주택)·계약일(아파트) 구분.
- **근거**: 산정기준이 다른 수치를 한 시계열에 섞으면 오해. `docs/data-crawl-prompt.md`에 표준 metric 고정.

## D14. 카테고리 2축 + 지표 출처군 세분
- **결정**: 유형(뉴스/지표) × 분야(정책·규제/시장·실거래/금리·거시). 지표는 출처군으로 세분 — 부동산원·국토부 동향 / KB·실거래가 / 한은·금융위 거시.
- **근거**: 사용자가 제안한 "뉴스/정책/실거래동향/실거래가격동향"의 정규화.

## 운영 결정(Workflow)
- **W01 배포 워크플로**: 피처 브랜치 작업 → 푸시 → (강제푸시 차단 우회) `merge -s ours origin/<branch>`로 origin 트립을 조상화 → squash-merge PR → main 푸시 시 Pages 자동 배포.
- **W02 병렬 에이전트 크롤/검증**: 데이터 수집·전수 검증을 백그라운드 서브에이전트로 병렬화. 지표 수집 시 **출처 없는 값은 생략·날조 금지** 원칙.
- **W03 컨테이너 휘발성**: 보존할 가치가 있는 모든 것(시드·리포트·결정·히스토리·프롬프트)은 커밋·푸시한다.
