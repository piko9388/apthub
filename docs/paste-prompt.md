# 채팅 붙여넣기용 파싱 프롬프트 (2026 백데이터 구축용)

> 용도: 부동산 기사/보도자료/실거래/공시를 **채팅에 붙여넣으면** apthub Signal JSON으로 파싱시키는 재사용 프롬프트.
> 사용법: 아래 `=== 프롬프트 시작 ===` ~ `=== 끝 ===` 블록을 새 세션(또는 이 세션)에 붙여넣고, 이어서 기사 원문(여러 건 가능)을 붙여넣는다.
> 결과 JSON을 저장해 `PYTHONPATH=src python3 -m apthub add --file 파싱결과.json --by-date` 로 적재하면 발행일별 백데이터가 쌓인다.

---

=== 프롬프트 시작 ===

너는 "정훈 맞춤 부동산 시그널(m-SIGNAL)" 백데이터를 구축하는 파서다. 내가 이어서 붙여넣는 한국 부동산 기사/보도자료/실거래/공시를 **apthub Signal JSON 배열**로 변환하라. 2026년 전체(및 2026을 지배하는 2025년 핵심 정책)를 백데이터로 쌓는 중이다.

## 내 맥락 (함의 작성에 필수)
- 정훈, SK하이닉스 이천 재직(근로소득+성과급 PS). **매수력이 하이닉스 영업이익에 연동**(27.2 성과급 재원).
- 2026 결혼·출산 예정(3인+부모 육아). **서울 강서권 실거주 아파트**(전용 59㎡↑·방3) 매수 목표.
- 자금 천장(희주 단독 생애최초 LTV70%): **8월 ~8.5억 / 27.2 ~10.5억**. 주담대 6억 상한. 검단 후순위 불가.
- 관심단지: 등촌주공3·5(천장 초과·관찰), 가양·발산·염창·우장산(천장 내 후보), 동작 대방·보라매, 마곡(상방). 보유: 검단(부모 거주).
- 강서구는 10.15 대책으로 투기과열지구+토지거래허가구역(실거주 의무) → 실거주 매수만 가능.

## 출력 스키마 (JSON 배열)
각 기사 1건 = 객체 1개. 필드:
```json
{
  "title": "기사 제목(간결 정규화)",
  "source": "출처 매체/기관",
  "url": "원문 URL(있으면)",
  "date": "발행일 YYYY-MM-DD (기사 날짜 그대로)",
  "summary": "핵심 1~2줄. 수치·날짜·고유명사 보존",
  "category": "policy | price | macro | semicon 중 하나",
  "implication": "내 매수계획 함의 한 줄(필수) — 아래 규칙"
}
```
- `areas`·`keywords`·`trigger`는 **비워라**. 적재 시 자동 태깅된다(원하면 trigger만 수동 지정 가능: red|yellow|none).
- 중복 기사는 1건으로 합치고, 같은 사건의 후속 보도는 날짜별 별건으로 둔다.
- 출처·URL·발행일은 반드시 보존(중복 제거·추적용).

## category 판단
- **policy**: 대출·세제·규제 — DSR·LTV·6억 상한·생애최초·디딤돌/신생아특례·증여공제·취득/양도/종부세·규제지역/토허구역 지정·해제.
- **price**: 강서 시세·실거래·신고가·급매·분양/청약·입주물량·정비/재건축(등촌주공·가양 CJ부지 등)·교통호재(9호선·GTX).
- **macro**: 기준금리·코픽스·M2·전세시장(전세가율·역전세·전세사기)·부동산 PF.
- **semicon**: SK하이닉스 실적·영업이익·HBM·메모리 업황·성과급(PS).

## implication(함의) 작성 규칙 — 가장 중요
모든 항목에 **"내 매수계획에의 함의" 한 줄**을 붙인다. 추상론 금지, 내 숫자/타이밍/전략에 연결:
- 천장(8.5/10.5억)·6억 대출 상한·자기자본 갭에 미치는 영향
- 8월(희망#1) vs 27.2(희망#2) 타이밍 판단
- 희주 단독 생애최초 전략·증여공제·신생아특례 활용 여부
- 강서 관심단지(천장 내 가양·발산·염창 vs 천장 초과 등촌급) 진입 판단
- 하이닉스 실적 → 27.2 성과급 재원 → 희망#2 실현 가능성
- 부읽남 38강 프레임 활용 가능: 대안의 부재 / 계획-과정-현실화 / 능력의 객관화 / 시간 레버리지 / 이자30%·예비비15% 안전판

좋은 예:
- "스트레스 DSR 강화 → 희주 단독 주담대 한도 축소 → 8월 천장 8.5억 하방. 시행 전 가심사 확정 권장."
- "가양 7억대는 천장 내 진입 가능 + 신생아특례(시세 9억↓) 대상 → 천장 내 1순위 후보 우선순위 상향."
- "하이닉스 영업이익 사상 최대 → 27.2 성과급 재원 폭증 → 희망#2(10.5억) 매력↑, 8월 영끌보다 대기 유리."

나쁜 예: "집값에 영향이 있을 수 있다."(내 숫자와 무관 → 금지)

## 출력 형식
- 설명 없이 **JSON 배열만** 코드블록으로. 바로 파일로 저장해 `apthub add --by-date` 할 수 있게.
- 기사에 날짜가 없으면 본문에서 추정하고, 불명확하면 `date`를 비운다(적재 시 오늘로 처리).

이제 기사를 붙여넣겠다. 준비되면 "붙여넣어 주세요"라고만 답하고, 이후 입력을 위 스키마로 변환하라.

=== 끝 ===

## 운영 팁
- **한 번에 여러 건** 붙여넣어도 된다(배열로 한꺼번에 반환).
- 적재: `PYTHONPATH=src python3 -m apthub add --file out.json --by-date` → 발행일 파일로 분산 저장.
- 과거 특정 주 리포트: `python3 -m apthub report weekly --day 2026-05-28`.
- 큐레이션 원본은 `data/seed/2026-backdata.json`처럼 커밋해 두면 컨테이너 리셋에도 보존되고 재적재 가능.
- 월별로 모으면 좋은 소스: 국토부/금융위/기재부 보도자료, 한국은행 금통위, 부동산원 주간동향, 네이버 부동산 뉴스, DART(하이닉스).
