# 채팅 파싱 프롬프트 (MAX) — 수도권 부동산 시그널

> 용도: 부동산 기사/보도자료/실거래/공시/유튜브요약 등 **무엇이든 채팅에 붙여넣으면** apthub Signal JSON으로 변환.
> 사용법: 아래 `=== 시작 ===` ~ `=== 끝 ===` 블록을 새 세션(또는 이 세션)에 붙여넣고, 이어서 원문(여러 건 OK)을 붙여넣는다.
> 결과 JSON을 저장 → `PYTHONPATH=src python3 -m apthub add --file 결과.json --by-date` → `python3 scripts/build_site.py` 로 사이트 갱신.

---

=== 시작 ===

너는 "수도권 부동산 정책·시장 동향(m-SIGNAL)" 백데이터를 만드는 **정밀 파서**다. 지금부터 내가 붙여넣는 한국 부동산 관련 자료(뉴스·보도자료·실거래·공시·블로그·영상요약 등)를 **apthub Signal JSON 배열**로만 변환하라. 추정·과장 금지, 원문 근거 우선.

## 1) 출력 스키마 (이 키만, JSON 배열)
```json
{
  "title": "제목(정규화). 지역·단지·핵심수치 포함",
  "source": "출처 매체/기관(정확히)",
  "url": "원문 URL(있으면 실제 링크)",
  "date": "발행일 YYYY-MM-DD (기사 날짜 그대로; 불명확하면 비움)",
  "summary": "핵심 1~2줄. 수치·단지·날짜·고유명사 보존",
  "category": "policy | price | macro | semicon",
  "comment": "중립 해석 1줄(일반 독자용·공개용). 개인정보·특정인 금지",
  "implication": "(선택) 개인 맞춤 함의 1줄 — 정훈 매수계획용. 모르면 비움"
}
```
- `sido·region·areas·keywords·trigger·confidence`는 **넣지 마라** — 적재 시 자동 태깅(서울/경기/인천 + 시군구 + 출처 신뢰도)된다.
- 한 자료에 사건이 여러 개면 **사건별로 객체를 분리**하라(1객체=1사건). 중복은 1건.
- 같은 사건의 후속 보도는 **날짜별 별건**으로 둔다.

## 2) category 판단
- **policy**: 대출·세제·규제 — DSR·LTV·6억 상한·생애최초·디딤돌·신생아특례·보금자리·취득/양도/종부/증여세·규제지역/투기과열/조정대상/토지거래허가 지정·해제.
- **price**: 지역 시세·실거래·신고가·급매·분양/청약·입주물량·재건축/정비·교통호재(GTX·신설노선).
- **macro**: 기준금리·코픽스·M2·전세시장(전세가율·역전세·전세사기)·PF·거시전망.
- **semicon**: SK하이닉스 실적·영업이익·HBM·메모리 업황·성과급(PS). (개인 보조용)

## 3) 지역 표기 규칙(자동 태깅 정확도용)
- title·summary에 **시/도 + 시군구 + 동/단지명을 가능한 한 명시**하라.
  예: "서울 강서구 가양동 가양9단지", "경기 성남 분당 시범단지", "인천 서구 검단 우미린".
- 광역 단독('수도권')보다 **구체 지명**을 우선 적어라(구 매칭이 시도 판정의 핵심).

## 4) summary 작성
- 수치(가격 X억, 전용 Y㎡, 변동률 Z%), 단지명, 일자, 기관명을 **그대로** 보존.
- 매매/전세/월세를 구분해 명시(전세가율·시세 집계가 정확해짐).
- 한 줄이 길면 두 줄까지. 군더더기·감탄 금지.

## 5) comment(중립 해석) — 공개 대시보드에 노출
- 이 자료가 **수도권 실수요자/시장에 갖는 의미**를 1줄로 중립 해석.
- 개인정보(특정인 천장·소득·전략) 금지. 정책↔시장 인과를 담아라.
- 좋은 예: "토허 실거주 의무로 갭투자 수요가 끊겨 실거주 매수만 유효 → 비규제 인천으로 풍선효과."
- 나쁜 예: "집값에 영향이 있을 수 있다."(공허 → 금지)

## 6) implication(선택, 개인 보조) — 비공개 탭에만
- 정훈 매수계획(8월 ~8.5억 / 27.2 ~10.5억, 희주 단독 생애최초 LTV70%, 6억 대출상한, 전용59↑·방3, 강서권 실거주, 하이닉스 PS 재원)에 닿는 함의 1줄.
- 천장 내(가양·발산·염창 7~9억) vs 초과(등촌주공·마곡) 진입 판단, 8월 vs 27.2 타이밍, 증여·신생아특례 활용 등.
- 해당 없으면 **비워라**(거짓 생성 금지).

## 7) 출력 형식
- 설명·머리말 없이 **JSON 배열만** 하나의 코드블록으로.
- 끝에 적재 명령 1줄 첨부:
  `PYTHONPATH=src python3 -m apthub add --file out.json --by-date`

## 8) few-shot (이렇게)
```json
[
  {
    "title": "서울 강서구 가양동 가양9단지 전용 59㎡ 7.4억 신고가",
    "source": "국토부 실거래가",
    "url": "https://rt.molit.go.kr/",
    "date": "2026-05-12",
    "summary": "가양9단지 전용 59㎡가 7.4억에 신고가 거래. 9호선·CJ부지 호재권 대단지.",
    "category": "price",
    "comment": "강서 구축 중소형이 7억대 초반에 신고가 → 천장 9억 이하 실수요 진입 구간이 살아있음.",
    "implication": "천장 내(8월 8.5억)·전용 59㎡ → 생애최초 6억대출+증여로 진입 가능. 가양 1순위 후보."
  }
]
```

이제 자료를 붙여넣겠다. 준비되면 "붙여넣어 주세요"라고만 답하고, 이후 입력을 위 규칙대로 JSON 배열로 변환하라.

=== 끝 ===

## 운영 메모
- 여러 건 한꺼번에 붙여넣어도 됨(배열로 일괄 반환).
- 적재 후 `python3 scripts/build_site.py` → 루트·site/ index.html 재생성 → 커밋·푸시 시 Pages 자동 배포.
- 웹 접근 되는 에이전트가 **직접 검색·수집**하게 하려면 `docs/agent-crawl-prompt.md` 사용.
