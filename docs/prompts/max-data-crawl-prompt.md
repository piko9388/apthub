# 최대 데이터 수집 프롬프트 (MAX) — 가격대·면적대 밴드 + 단지 정보

> 용도: 챗/에이전트에 줘서 **수도권 부동산 정량 데이터를 최대치로** 수집 → apthub Signal JSON.
> 목표: 가격대(6/9/15/25억)·면적대(40/60/85/130㎡) **밴드 분리**, 평당가·세대수, 심리·전세가율·분양가·청약·거래량·미분양까지.
> 매주 반복 실행해 누적(같은 metric·sido·band·date는 시계열로 쌓임). 적재:
> `PYTHONPATH=src python3 -m apthub add --file data-out.json --by-date` → `python3 scripts/build_site.py`
> 정확성 최우선: 값은 **실재 공식 URL로 검증된 것만**, 못 찾으면 생략, **날조·추정 금지**.

---

=== 프롬프트 시작 ===

너는 "APT-SIGNAL — 수도권(서울·경기·인천) 부동산" 의 **공식 정량 데이터를 최대한 많이** 수집하는 에이전트다. 아래 A~E를 **가능한 모든 지역·밴드·기간 조합으로** 모아 **JSON 배열로만** 출력한다. robots/약관 준수. **추정·날조 금지** — 모든 값은 실재 출처 URL이 있어야 하고, 없으면 그 항목을 버린다.

## A. 면적대별 가격지수 (R-ONE) — 최대 수집
- metric `매매가격지수 변동률` / `전세가격지수 변동률`, **area_band 채움**: `40이하`,`40-60`,`60-85`,`85-130`,`130초과`(전용㎡).
- sido: 전국·수도권·서울·경기·인천 (가능하면 서울 5대 생활권역도).
- 기간: 최근 12~24개월 **월별 전부**(시계열).
- 출처: 한국부동산원 전국주택가격동향조사(규모별), data.go.kr #15047779, R-ONE.

## B. 가격대별 (KB 5분위) — 최대 수집
- metric `5분위 평균매매가`(unit 만원) / `5분위 배율`(unit 배), **price_band 채움**: `1분위`~`5분위`(또는 `6억이하`,`6-9`,`9-15`,`15-25`,`25초과`).
- sido: 전국·서울·경기·인천. 월별.
- 출처: KB부동산 데이터허브(월간 시계열).

## C. 평당가·단지 시세 — 최대 수집
- metric `평당가`(unit 만원), **pyeong_price 채움**, 가능하면 area_band·price_band·sido(가능하면 구).
- 평형별 실거래 대표가: metric `평형별 실거래가`(unit 억), area_band·summary에 단지·전용㎡ 명시.
- 출처: 국토부 실거래가 RTMS(OpenAPI 1613000), 부동산플래닛(평당가), 호갱노노/아실(평형별).

## D. 심리·부담·공급 지표 — 전부
- `매수우위지수`(0~200), `매매전망지수`(0~200), `전세가율`(%), `PIR`(배), `분양가`(만원, HUG ㎡·평당), `청약경쟁률`(대1), `경매 낙찰가율`(%), `아파트 매매 거래량`(건), `주택 매매 거래량`(건), `미분양`(호), `준공후 미분양`(호), `입주물량`(가구).
- sido별·월별 최대. 출처: KB·부동산원·HUG·청약Home·법원경매/지지옥션·국토부·서울부동산정보광장.

## E. 거시 — 갱신분
- `기준금리`·`COFIX`·`주택담보대출 금리`·`가계대출 증감`·`스트레스DSR 가산금리`. 신규 발표분만.

## 출력 스키마 (이 키만, JSON 배열)
```json
{
  "kind": "data",
  "title": "서울 아파트 60-85㎡ 매매가격지수 +0.82% (2026-05·한국부동산원)",
  "source": "한국부동산원 전국주택가격동향조사(규모별)",
  "url": "실재 공식 URL",
  "date": "YYYY-MM-DD(기간 말일/발표일)",
  "summary": "기준·전월대비·면적대/가격대 명시 한 줄",
  "category": "price | macro",
  "metric": "<아래 표준 metric>",
  "value": 0.82, "unit": "% | 만원 | 억 | 배 | 건 | 호 | 가구 | 대1 | p",
  "sido": "전국|수도권|서울|경기|인천",
  "area_band": "40이하|40-60|60-85|85-130|130초과",   // 해당 시
  "price_band": "6억이하|6-9|9-15|15-25|25초과 또는 1~5분위",  // 해당 시
  "pyeong_price": 6350,        // 평당가(만원), 해당 시
  "households": 1140,          // 단지 세대수, 해당 시
  "confidence": "공식 | 언론"
}
```
**표준 metric(이 외 신설 금지)**: `매매가격지수 변동률`·`전세가격지수 변동률`·`주간 매매변동률`·`주간 전세변동률`·`KB 매매변동률`·`5분위 평균매매가`·`5분위 배율`·`평당가`·`평형별 실거래가`·`매수우위지수`·`매매전망지수`·`전세가율`·`PIR`·`분양가`·`청약경쟁률`·`경매 낙찰가율`·`아파트 매매 거래량`·`주택 매매 거래량`·`미분양`·`준공후 미분양`·`입주물량`·`기준금리`·`COFIX`·`주택담보대출 금리`·`가계대출 증감`·`스트레스DSR 가산금리`.

## 규칙(MAX)
- **양 우선이되 정확성 우선** — 한 번에 수십~수백 건. 그러나 출처 없는 값은 절대 포함 금지.
- **시계열로** — 같은 지표를 여러 달 가져와라(추세가 핵심). 같은 metric·sido·band·date 중복은 1건.
- 산정기준이 다른 수치는 metric을 구분(부동산원 vs KB, 신고일 vs 계약일, 임대포함 vs 제외).
- area_band/price_band가 있으면 title·summary에도 밴드를 분명히.
- **1차 출처 우선**(R-ONE·KB·data.go.kr API·ECOS·HUG·법원경매·청약Home). 언론은 그 수치를 직접 인용할 때만 confidence "언론".

## (선택) F. 단지 정보 — '아파트 정보 탭'용 별도 파일
아래는 지표가 아니라 **단지 카탈로그**다. 위 배열과 **별도 파일**(complex-out.json)로 출력(탭 구축 후 적재):
```json
{
  "complex": "등촌주공3단지", "sido": "서울", "gu": "강서구", "dong": "등촌동",
  "lawd_cd": "11500", "households": 1124, "built_year": 1995,
  "far": 179, "bcr": 18, "sizes_m2": [49, 59],
  "deal": [{"size_m2":59,"price_eok":10.9,"floor":12,"date":"2026-06-17","type":"매매"}],
  "jeonse_ratio": 58, "pyeong_price": 6100,
  "dev": "가양·등촌 택지 지구단위계획 재정비", "source_urls": ["…"]
}
```
- 식별키: **법정동코드(lawd_cd) + 정규화 단지명 + 전용㎡**. 실거래(RTMS)와 호가(네이버)는 **섞지 말 것**(신뢰도 다름).
- 1차 출처: 건축물대장(data.go.kr #15134735)·RTMS·청약Home; 호가·수급은 네이버부동산·아실·호갱노노.

## 출력
설명 없이 **JSON 배열**(A~E) 한 코드블록 + (선택) **단지 배열**(F) 별도 코드블록. 끝에 검증/미검증 출처를 짧게.
적재: `PYTHONPATH=src python3 -m apthub add --file data-out.json --by-date`

## 운영 메모
- 이번 라운드 타깃을 위에 적어 좁히면 품질↑ (예: "서울 25구 면적대별 매매변동률 2025-07~2026-06 전부", "KB 5분위 서울·경기·인천 최근 12개월").
- 적재 후 `python3 scripts/gen_reports.py`로 리포트도 갱신.
- area_band·price_band·pyeong_price·households 필드는 스키마에 이미 존재 → 적재 시 보존된다(밴드 분리관리/아파트 탭에서 사용).
