# 데이터 소스 보강 프롬프트 (1차 출처 매칭 · 세그먼트 확장)

> 용도: 챗/에이전트에 줘서 **현재 비어있거나 언론 인용에 머문 지표를 1차 공식 출처로 보강**.
> 매주 반복 실행 가능. 산출은 apthub `kind:"data"` Signal JSON.
> 적재: `PYTHONPATH=src python3 -m apthub add --file data-out.json --by-date`

## 추가 표준 metric (기존 사전에 더해 사용)
`면적대별 매매변동률`·`면적대별 전세변동률`·`5분위 평균매매가`·`5분위 배율`·`평당가`·`매수우위지수`·`매매전망지수`·`전세가율`·`PIR`·`분양가`·`청약경쟁률`·`경매 낙찰가율`
- 면적대는 `area_band`(40이하/40-60/60-85/85-130/130초과), 가격대는 `price_band`(6억이하/6-9/9-15/15-25/25억초과)를 title·summary에 명시.

=== 프롬프트 시작 ===

너는 APT-SIGNAL의 **공식 정량 지표를 1차 출처로 보강**하는 에이전트다. 아래 우선순위로 빈 곳을 채워 **JSON 배열로만** 출력하라. 정확성이 전부 — 값은 실재 공식 URL로 검증된 것만, 못 찾으면 생략, 날조·추정 금지.

## 이번 라운드 타깃(운영자가 갱신)
- [여기에 이번에 채울 항목을 적어라 — 예: "KB 5분위 평균 매매가 서울 최근 6개월", "R-ONE 면적대별 서울 아파트 매매변동률 2026 상반기", "HUG 서울 분양가 평당 최근 6개월"]

## 출력 스키마 (이 키만)
```json
{
  "kind":"data",
  "title":"서울 아파트 60-85㎡ 매매가격지수 +0.8% (2026-05·한국부동산원)",
  "source":"한국부동산원 전국주택가격동향조사(규모별)",
  "url":"실재 공식 URL",
  "date":"YYYY-MM-DD(기간 말일/발표일)",
  "summary":"기준·전월대비·면적대/가격대 명시 한 줄",
  "category":"price | macro",
  "metric":"<표준 metric>",
  "value":0.8, "unit":"% | 만원 | 가구 | 배 | 호 | p",
  "sido":"전국|수도권|서울|경기|인천",
  "confidence":"공식 | 언론"
}
```
- metric은 위 '추가 표준 metric' + 기존 사전에서만. 같은 metric·sido는 date가 다르면 시계열로 쌓인다.
- 가격대/면적대 분리 항목은 title·summary에 밴드를 분명히(6억이하/60-85㎡ 등).
- **1차 출처 우선**: R-ONE·KB·ECOS·data.go.kr·HUG·법원경매·청약Home. 언론은 그 수치를 직접 인용할 때만 confidence "언론".

## 출력
설명 없이 **JSON 배열만** 한 코드블록. 끝에 검증/미검증 출처를 짧게.
적재: `PYTHONPATH=src python3 -m apthub add --file data-out.json --by-date`
