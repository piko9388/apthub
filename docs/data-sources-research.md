# 데이터 소스 수집 방식 조사 (Phase 1 기준)

> 00-prompt.md 과제 (a): 소스별 수집 가능 방식(Open API / RSS / 크롤 / 불가)을 정리.
> 실제 키 발급·필드 확인은 Phase 2 구현 시 검증. 아래는 2026.1 기준 조사.

## 우선순위 요약

| 등급 | 의미 | 해당 소스 |
|---|---|---|
| ⭐⭐⭐ | Open API(키 발급, 가장 안정·합법) | RTMS, 한국부동산원 R-ONE, ECOS, DART |
| ⭐⭐ | RSS/검색 API | 경제지·포털 부동산 뉴스 |
| ⭐ | 공식 페이지 크롤(보도자료 목록, robots 준수) | 국토부/금융위/기재부/한은 보도자료, 청약홈 |
| ⛔ | 약관상 자동수집 제한·우회 위험 | KB부동산 시세 스크래핑, 네이버부동산 매물 |

## 정책 / 제도

| 소스 | 대상 | 권장 방식 | 비고 |
|---|---|---|---|
| 국토교통부 보도자료 | 부동산 대책·규제지역·공급 | 페이지 목록 크롤(저빈도) | RSS 불안정. 제목·링크·일자만 |
| 금융위원회 | DSR·LTV·대출규제 | 보도자료 페이지 크롤 | 6.27 대책류 핵심 |
| 기획재정부 | 세제(취득·양도·종부·증여) | 보도자료 페이지 크롤 | 혼인·출산 증여공제 추적 |
| 한국은행 | 기준금리·통화량 | **ECOS Open API** | 금통위 결정문은 페이지 |
| 법제처 입법예고 | 시행령·규칙 개정예고 | 국가법령정보 Open API/페이지 | 시행 전 선행지표 |

## 시세 / 실거래 / 청약

| 소스 | 대상 | 권장 방식 | 비고 |
|---|---|---|---|
| 국토부 실거래가(RTMS) | 아파트 실거래 | **Open API**(공공데이터포털) | 핵심. 법정동코드+단지명 매핑 필요 |
| 한국부동산원 R-ONE | 주간/월간 시세·통계 | **Open API**(R-ONE) | KB 대안. 주간변동률 |
| KB부동산 | KB시세·주간동향 | 주간 통계 파일/보고서 | 시세 스크래핑은 약관 주의 |
| 청약홈(APPLYHOME) | 분양·청약 일정·경쟁률 | 공공데이터포털 청약 API/페이지 | 강서 분양 모니터링 |

| 부동산 유튜브/블로그 | 해설·관점 | 채널 RSS | 부읽남 등(판단 프레임 보조) |

## 반도체 / 소득 연동

| 소스 | 대상 | 권장 방식 | 비고 |
|---|---|---|---|
| DART(전자공시) | SK하이닉스 실적·공시 | **Open API**(DART) | 분기 실적·주요사항보고 |
| 반도체 전문 뉴스 | HBM·메모리 업황 | RSS/검색 | 성과급 재원 신호 |

## 수집 원칙 (재확인)

- 공공데이터포털·ECOS·DART·R-ONE은 **API 키 발급 후** 사용(가장 안정·합법).
- 뉴스는 RSS 우선, 없으면 검색 API. 본문 전체보다 제목·요약·링크.
- 출처 URL·발행일시 보존, 중복 제거(URL/제목 해시 — `Signal.id`).
- robots.txt·이용약관 준수, 무단 대량 크롤 금지. 저빈도·캐시 우선.
