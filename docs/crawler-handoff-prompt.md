# 부동산 뉴스·정책 크롤러 + 데일리/위클리 리포트 — 핸드오프 프롬프트

> 다른 코드방/채팅방에서 "정훈 맞춤 m-SIGNAL"을 만들기 위한 **그대로 복사용 프롬프트**.
> wealth-hub(부동산 의사결정 시스템)에서 합의된 맥락·필터·트리거를 압축해 담았다.
> 아래 `=== 프롬프트 시작 ===` ~ `=== 프롬프트 끝 ===` 사이를 통째로 새 세션에 붙여넣으면 된다.

---

## 이 문서의 사용법
1. 아래 프롬프트 블록을 복사 → 새 코드/채팅방에 붙여넣기.
2. 첨부로 같이 넘기면 좋은 파일(선택): `docs/budreadnam-lectures.md`(판단 프레임), `data/processed/finance-model.json`(내 자금/규제 SSOT).
3. 크롤러는 **개인정보(급여·자산) 없이** 시장 데이터만 다루므로 public 환경에서 작업해도 안전. (단 finance-model.json은 민감 → 넘길 때 주의)

---

=== 프롬프트 시작 ===

너는 나(정훈)를 위한 **부동산 뉴스·정책·시장 시그널 크롤러 + 데일리/위클리 리포트** 시스템을 만드는 엔지니어다. "m-SIGNAL" 같은 개인 맞춤 부동산 브리핑이 목표다.

## 1. 나의 상황(왜 이 데이터가 중요한가)
- SK하이닉스 이천 재직(근로소득 + 성과급 PS). **내 매수 능력은 반도체 업황·하이닉스 영업이익에 연동**된다 → 반도체/하이닉스 뉴스도 모니터링 대상.
- 2026년 8월(희망#1) 또는 2027년 2월 성과급 후(희망#2) **서울 강서권 실거주 아파트 매수** 목표.
- 명의 전략: **희주(배우자) 단독 생애최초(LTV70%)가 1순위**(정훈 무주택 보존 → 향후 추가 매수 채널). 공동명의는 2순위.
- 현재 확정 천장(가용 자금 기준): **8월 약 8.5억 / 27.2 성과급 후 약 10.5억**(희주 단독). 검단 후순위 1억은 불가(아빠 신용)로 제외됨.
- 따라서 **대출 규제(DSR·LTV·6억 상한)·금리·생애최초 요건 변화**가 내 매수 가능액을 직접 좌우한다 → 최우선 모니터링.

## 2. 모니터링 대상 (우선순위順)
**A. 대출/세제 정책 (최우선)**
- 스트레스 DSR 단계·산정방식 변경, DSR 한도(40%) 조정
- 주담대 한도(현 수도권 6억 상한), LTV(규제지역 무주택 40%, 생애최초 70%) 변경
- 생애최초·신생아 특례·디딤돌/보금자리론 요건·금리
- 취득세·양도세·종부세, 혼인·출산 증여공제(결혼/출산 예정이라 직접 해당)
- 규제지역(투기과열·조정대상) 지정/해제 — 특히 **서울 강서구, 인천 서구(검단)**

**B. 관심 지역 시세·공급 (강서 중심)**
- 서울 강서구: **등촌주공3·5단지, 가양, 발산, 염창, 우장산**
- 인접: 동작 대방·보라매, 마곡
- 검단(인천 서구 원당동) — 보유 자산(e편한세상 웰카운티)
- 실거래가(국토부 RTMS), KB/부동산원 주간시세, 입주물량, 분양/청약, 재건축·정비사업 진행(등촌주공 정비, 가양 CJ부지 스타필드, GTX·9호선 호재)

**C. 거시·금리**
- 한국은행 기준금리, 주담대 금리(코픽스), 통화량(M2)
- 부동산 PF·전세시장(역전세·전세사기 동향)

**D. 반도체/소득 연동**
- SK하이닉스 실적·영업이익 전망, HBM/메모리 업황, 성과급 관련 뉴스(내 27.2 성과급 재원)

## 3. 개인화 필터 / 트리거 (리포트에서 강조할 것)
- **🔴 즉시 알림 트리거**: DSR·LTV·6억 상한 변경 / 생애최초 요건 변경 / 강서·검단 규제지역 변동 / 강서 관심단지 신고가·급매 / 한은 기준금리 변경
- **🟡 주목**: 등촌주공 정비사업 단계 진전, 강서 입주물량·분양, 하이닉스 실적 서프라이즈
- 각 뉴스에 **"내 매수계획에의 함의"** 한 줄을 붙일 것 (예: "DSR 강화 → 희주 단독 주담대 5.5억 한도 축소 가능 → 8월 천장 하락 리스크").
- 판단 프레임은 첨부 `budreadnam-lectures.md`(부읽남 38강)의 원칙 사용: 대안의 부재, 계획-과정-현실화 3단계 가격변화, 능력의 객관화, 시간 레버리지, 이자30%·예비비15% 안전판.

## 4. 데이터 소스 (제안)
- **정책**: 국토교통부·금융위·기획재정부 보도자료, 한국은행, 법제처(입법예고)
- **시세/실거래**: 국토부 실거래가 공개시스템(RTMS) Open API, 한국부동산원 R-ONE API, KB부동산
- **청약**: 청약홈(applyhome)
- **뉴스**: 네이버 뉴스 부동산 섹션, 주요 경제지(한경·매경·이데일리 부동산), 부동산 카페/유튜브(부읽남 등) RSS
- **반도체**: 전자공시(DART) SK하이닉스, 반도체 뉴스
- 가능하면 **RSS/Open API 우선**(robots.txt·이용약관 준수). 무단 대량 크롤링 회피.

## 5. 산출물 포맷
**데일리 (평일 아침)**
```
📅 [날짜] 정훈 부동산 시그널 — Daily
🔴 핵심 트리거 (있으면): …  ← 내 매수계획 함의 1줄
📊 시장: 강서 관심단지 실거래/호가 변동, 금리
📰 정책/뉴스 Top 3 (각 1줄 요약 + 내게 미치는 영향)
💡 오늘의 액션(있으면): 가심사 문의/매물 체크 등
```
**위클리 (일요일)**
```
🗓️ [주차] 정훈 부동산 시그널 — Weekly
1. 이번 주 정책 변화 & 내 천장(8.5/10.5억)에의 영향
2. 강서 관심단지 주간 시세 표(등촌주공3·5/가양/발산/염창/대방)
3. 금리·거시·반도체(하이닉스) 요약
4. 부읽남 관점 코멘트(이번 주 시장을 38강 원칙으로 해석)
5. 다음 주 체크포인트 / 액션 아이템
```

## 6. 기술 요건
- 스택 자유(Python 권장: requests/httpx + feedparser + BeautifulSoup, 스케줄러 cron/GitHub Actions).
- 수집 → 정규화(JSON) → 필터(관심 키워드·지역·트리거) → 요약(LLM, 최신 Claude 모델 권장) → 리포트(Markdown/HTML/메일·텔레그램 발송).
- 중복 제거·출처 링크 보존·날짜 정규화. 수집 원문은 캐시.
- **로컬/프라이빗 우선**(내 관심사·계획은 민감). 발송은 개인 메일/텔레그램.

먼저 **(a) 데이터 소스별 수집 가능 방식(API/RSS/크롤) 조사 + (b) MVP 아키텍처(수집→필터→요약→리포트) 설계**부터 제안해줘. 그 다음 데일리 리포트 생성기를 최소기능으로 구현하자.
